# codex.md

Repo-specific guidance for Codex / OpenAI agents. Read before editing.

## Project at a glance

Lightweight Windows screenshot/burst-capture tool. Targets faster cold start than the built-in Snipping Tool. Three build paths:

| Build | Script | Output | Use case |
|---|---|---|---|
| Native Win32 (C) | `build-native.ps1` | `build\WinCapNative.exe` (~30 KB) | Fast cold start. Primary target. |
| .NET Framework | `build.ps1` | `build\WinCapSimple.exe` (~37 KB) | Builds via system `csc.exe`, no SDK. |
| .NET 10 R2R | `build-dotnet.ps1` | `build\dotnet-r2r\WinCapSimple.exe` | Requires .NET 10 SDK. |

Native build prerequisite: Visual Studio Build Tools (`Microsoft.VisualStudio.Component.VC.Tools.x86.x64`).

## Self-tests

Run after capture-path changes:

```powershell
.\build\WinCapNative.exe --self-test
.\build\WinCapNative.exe --self-test-dxgi
.\build\WinCapSimple.exe --self-test
.\build\dotnet-r2r\WinCapSimple.exe --self-test
```

Exit codes: `0` pass, `1` capture/save failed, `2` DXGI path did not run or second burst failed. `--self-test-dxgi` runs display burst twice in one process to verify the D3D11 device cache.

## Invariants — do not break

### Native (`native/wincap_native.c`)

- `/NODEFAULTLIB` + custom `wWinMainCRTStartup` entry. **No CRT headers** (`<stdio.h>`, `<string.h>`, `<stdlib.h>`, etc.). Hand-rolled `memset`/`memcpy` registered via `#pragma function` is required by intrinsic lowering — do not delete.
- DLL loading must use `LoadLibraryExW(..., LOAD_LIBRARY_SEARCH_SYSTEM32)`.
- Output is BMP. Do not introduce PNG/codec dependencies in native.
- Binary size baseline ~30 KB. Verify after touching `build-native.ps1` flags. Current flags: `/O1 /Os /GS- /GF /Gw /Gy /NODEFAULTLIB /OPT:REF /OPT:ICF /FILEALIGN:512`.

### Burst loop

- `AllocMem` (no zero) for buffers fully overwritten. `AllocZero` only when zero-init is required.
- `BurstFrameStore` is a single contiguous `block`; frame size must stay constant for the burst.
- `BeginBurstPerfScope` / `EndBurstPerfScope` bracket every burst loop. They handle `timeBeginPeriod(1)`, MMCSS `"Capture"`, and `HIGH_PRIORITY_CLASS`.
- `SleepUntilTick` uses `QueryPerformanceCounter` polling — do not replace with plain `Sleep`.

### DXGI display path

- Device/context/output1 cached in `g_dxgiCache`, keyed by `RECT bounds`. AddRef in `StoreDxgiDeviceCache`, Release in `ReleaseDxgiDeviceCache`.
- `AcquireNextFrame(0)` (zero timeout) is intentional. A positive timeout breaks fixed-interval recording.
- `DXGI_ERROR_WAIT_TIMEOUT` means "no new frame since last call"; reuse `lastFrame` and continue — it is not an error.
- DXGI accelerates only `MODE_DISPLAY`. Other modes use the GDI fallback in `CaptureBurstFrames`. Do not remove the fallback.
- Path is skipped on rotated displays.

### `CAPTUREBLT`

- Default off; toggled by the "Layered" UI button (`g_includeLayeredWindows` → `GetCaptureRop()`). Enabling by default adds a DWM sync round-trip to every `BitBlt`.

### Freeform mask

- `ApplyFreeformMaskWithNodes` is scanline polygon fill (sort + span). Do not regress to per-pixel `PointInsidePolygon` (the old O(W·H·N) version).
- Keep the `yi != yj` guard to avoid div-by-zero on horizontal edges.

### Managed builds (`src/`)

- `FastCaptureScope` deliberately does **not** set `GCLatencyMode.LowLatency` — it caused LOH pressure on long bursts. Use `GC.TryStartNoGCRegion` with explicit sizing if a no-GC region is required.
- Managed builds save PNG; native saves BMP. Intentional asymmetry.
- Clipboard copy publishes both `Bitmap` and DIB formats — see `ClipboardUtil.cs`.

## Editing patterns

### Capture-path optimization
1. Pick the build(s) to touch. Native is the fast path.
2. Run all four self-tests after the change.
3. Timing-sensitive changes (intervals, priorities) need a real burst test — self-tests run 1-frame bursts and won't catch jitter regressions.

### New UI control (native)
- Allocate a unique `IDC_*` ID (current range 1001–1014).
- Create HWND in `CreateMainControls`.
- Add to `LayoutMain` (the toolbar wraps when narrow — verify).
- Handle in `MainProc` `WM_COMMAND`.

### README edits
Bilingual (Korean + English) with parallel sections. Update both languages — do not let them drift.

## Out of scope (deliberately)

- **Windows Graphics Capture (WGC)**: not used. WinRT interop in `/NODEFAULTLIB` C is heavy; the win is window-mode correctness, not speed. If pursued, do it in the R2R build via `Windows.SDK.Contracts`.
- **DXGI for window/rectangle modes**: not implemented. Requires monitor-bounds detection + ROI crop + cross-monitor split handling.
- **Frame-skip / delta detection**: not implemented. Candidate next step if burst storage becomes an issue.
- **Tray-resident / pre-warm mode**: noted in README as the strongest cold-start option but not built.
- **WM_DISPLAYCHANGE handler**: not needed — DXGI cache invalidates naturally on bounds change, and failure self-heals.
